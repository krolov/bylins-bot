# sc.kerat.ru MUD client

Минимальный браузерный MUD-клиент на **Bun + TypeScript**.

Что уже заложено:

- Bun HTTP/WebSocket сервер
- TCP-прокси до удалённого MUD через `Bun.connect()`
- простой веб-интерфейс для наблюдения и ручных команд
- автоматическая отправка нескольких стартовых команд сразу после соединения
- автоподключение при старте приложения по встроенному startup-script
- готовый reverse-proxy конфиг для `sc.kerat.ru` с HTTP Basic Auth через Caddy

## Как это работает

Браузер подключается к вашему Bun-серверу по WebSocket. Сам Bun-сервер держит raw TCP-соединение с MUD и пересылает данные в обе стороны. Это нужно, потому что браузер сам по себе не умеет подключаться к MUD по обычному TCP/Telnet.

## Локальный запуск

```bash
cp .env.example .env
bun install
bun run start
```

После этого сервер сам поднимет MUD-сессию. `http://localhost:3000` можно открыть уже после старта — страница подключится к существующей сессии и покажет вывод.

`bun run start` сам собирает браузерный клиент перед запуском сервера.

## Режим разработки

```bash
bun run dev
```

## Настройки

Основные переменные лежат в `.env`:

- `MUD_HOST` — адрес MUD по умолчанию
- `MUD_PORT` — порт MUD
- `MUD_TLS` — нужен ли TLS
- `MUD_AUTO_CONNECT` — автоматически подключаться к MUD при каждом старте сервера
- `MUD_STARTUP_COMMANDS` — команды, которые отправляются автоматически после подключения
- `MUD_COMMAND_DELAY_MS` — пауза между стартовыми командами
- `MUD_LINE_ENDING` — `crlf` или `lf`

Стартовый сценарий подключения зафиксирован в `src/startup-script.ts`. По умолчанию он повторяет прошлую рабочую сессию: `bylins.su:7000` и стартовая команда `look`.

`MUD_STARTUP_COMMANDS` можно задавать через `;;`, например:

```env
MUD_STARTUP_COMMANDS=score;;inventory;;look
```

## Деплой на sc.kerat.ru

В репозитории есть:

- `Dockerfile`
- `deploy/docker-compose.yml`
- `deploy/Caddyfile`

Перед запуском:

1. направьте DNS `sc.kerat.ru` на ваш сервер;
2. создайте `.env` с реальными значениями;
3. сгенерируйте хэш пароля:

```bash
caddy hash-password --plaintext 'your-password'
```

4. подставьте хэш в `BASIC_AUTH_PASSWORD_HASH`;
5. запустите:

```bash
docker compose -f deploy/docker-compose.yml up -d --build
```

Caddy автоматически поднимет HTTPS для `sc.kerat.ru` и будет запрашивать логин/пароль перед доступом к приложению.

## Прогресс-бары HP и энергии

В топбаре отображаются два прогресс-бара: **HP** (красный) и **Энергия** (синий).

**Как работает:**

- Максимальные значения берутся из фразы, которую MUD присылает в ответ на команду `score`:
  ```
  Вы можете выдержать 50(50) единиц повреждения, и пройти 86(86) верст по ровной местности.
  ```
  Число в скобках — максимум. Запоминается сервером до следующего `score`.

- Текущие значения обновляются автоматически из каждой строки промпта:
  ```
  50H 86M 1421o Зауч:0 ОЗ:0 2L 5G Вых:СВЮЗ>
  ```

- После перезапуска сервера максимумы сбрасываются до 0. Нужно отправить `score`, чтобы бары заработали корректно.

**Перезапуск сервера** (после изменений в `src/server.ts`):

```bash
pm2 restart bylins-bot
```

Изменения в `public/styles.css` и `public/index.html` применяются без перезапуска — достаточно перезагрузить страницу в браузере.

Изменения в `src/client.ts` требуют пересборки клиента:

```bash
bun run build:client
```

## Ограничения текущей версии

- это именно простой текстовый клиент;
- telnet negotiation пока не обрабатывается отдельно;
- скриптовая система ещё не реализована, но сервер и клиент уже разделены так, чтобы её можно было добавить позже.
