# Скилл: Анализ лога MUD-зоны

Инструкция для агента — как по логу `mud-traffic.log` восстановить маршрут прохождения зоны, команды и NPC-взаимодействия.

---

## Входные данные

- Файл логов: `/var/log/bylins-bot/mud-traffic.log` (и `.log.1` за предыдущие сутки)
- Номер зоны (или диапазон vnum): например, зона 258 → vnum 25800–25899
- Персонаж игрока (опционально, для фильтрации)

---

## Шаг 1 — Определить диапазон vnum зоны

Зоны в Bylins MUD занимают диапазоны по 100 комнат:
- Зона 258 → vnum 25800–25899
- Зона 277 → vnum 27700–27799
- и т.д.

```bash
# Найти все упоминания комнат нужной зоны
grep -E "vnum=(258[0-9]{2})" /var/log/bylins-bot/mud-traffic.log | head -50
```

---

## Шаг 2 — Вытащить команды лидера (триггеры через `!`)

Лидер группы отправляет команды через `!` — бот их выполняет автоматически.

```bash
# Все исходящие команды бота
grep "direction=mud-out" /var/log/bylins-bot/mud-traffic.log

# Только триггеры от лидера (команды начинающиеся с !)
grep 'direction=mud-in' /var/log/bylins-bot/mud-traffic.log | grep '"!'
```

Важные команды для движения по зоне:
- `нырнуть в озеро` / `вынырнуть`
- `лезть дуб` / `лезть вниз`
- `пролезть кусты` / `раздвинуть ветки`
- `пробраться нора` / `вой пен`

---

## Шаг 3 — Найти NPC-взаимодействия

```bash
# Что говорили NPC (входящие сообщения с диалогами)
grep 'direction=mud-in' /var/log/bylins-bot/mud-traffic.log | grep -E '(говорит|сказал|шепчет|кричит)'

# Что отправляли NPC (квестовые реакции, награды)
grep 'direction=mud-in' /var/log/bylins-bot/mud-traffic.log | grep -E '(награда|кун|получаешь|дает|вручает)'

# Команды которые посылались NPC (г помогу, дать предмет и т.д.)
grep 'direction=mud-out' /var/log/bylins-bot/mud-traffic.log | grep -E '(^г |сказ |дать )'
```

---

## Шаг 4 — Найти мобов в зоне

```bash
# Мобы которых убивали (опыт, сообщения о смерти)
grep 'direction=mud-in' /var/log/bylins-bot/mud-traffic.log | grep -E '(убит|повержен|сражён|труп)'

# Фильтровать только по нужной зоне — смотреть сообщения рядом с vnum зоны
grep -E "vnum=(258[0-9]{2})" /var/log/bylins-bot/mud-traffic.log -A 5 | grep -E '(убит|повержен)'
```

---

## Шаг 5 — Восстановить порядок локаций

```bash
# Последовательность комнат в хронологическом порядке
grep -E "vnum=(258[0-9]{2})" /var/log/bylins-bot/mud-traffic.log | grep -oE "vnum=[0-9]+" | sort -u
```

Затем сгруппировать по смыслу:
- Повторяющиеся vnum = боевые комнаты (фарм)
- Уникальные vnum = транзитные комнаты или квестовые точки

---

## Шаг 6 — Собрать финальную строку

Формат вывода — **одна строка** в виде маршрута:

```
`спецкоманда` → убиваю [список мобов] → `спецкоманда` → [локация]: говорю [фраза NPC] → [результат/награда]
```

Правила:
- Только vnum из нужной зоны (фильтровать строго)
- Команды в обратных кавычках: `` `лезть дуб` ``
- NPC-фразы выделять: **`г помогу`**
- Без лишних деталей ("опасно", "осторожно" и т.д.)
- Стаб и базовый бой не упоминать — это само собой

---

## Пример результата (зона 258)

> `нырнуть в озеро` → убиваю рыбок + синюшную нежить, беру **голову водяной нежити** → `вынырнуть` → бью воинов/жителей/знахарей/охотников/ведуний/соглядатаев → `лезть дуб` → `дать голов духу` → `лезть вниз` → пролезаю кусты → **Седовласый старик** [25806]: говорю **`г помогу`** → бью карликов, подбираю **хрустальный шар** → отдаю шар старику → **5000 кун** → берег: рыбак/внук/пожилой рыболов.

---

## Частые ошибки

- **Не фильтруешь по зоне** — в лог попадают комнаты других зон (стаб, переходы). Всегда фильтруй строго по диапазону vnum.
- **Перепутал зоны** — vnum 27719 это не зона 258. Зона = первые 3 цифры vnum.
- **Включил команды из других зон** — триггеры `!` идут подряд, проверяй в каком vnum они выполнялись.
- **Пропустил квест** — квестовые NPC часто в отдельных комнатах (домик, горница). Искать по словам `сказал`, `просит`, `награда`.
