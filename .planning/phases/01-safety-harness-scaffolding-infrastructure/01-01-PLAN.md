Raw log line format:
```
[2026-04-18T12:34:56.789Z] session=voinmir-abc direction=mud-in message="...escaped JSON string..." key=value
```

Parser regex to use (from 01-PATTERNS.md):
```typescript
const LOG_LINE_REGEXP = /^\[(?<ts>[^\]]+)\] session=(?<session>\S+) direction=(?<direction>\S+) message=(?<message>".*")( .*)?$/;
```

Field semantics:
- `ts` — ISO-8601 timestamp (Z suffix always present)
- `session` — system OR profile UUID; "system" events are server-side lifecycle (per CONTEXT specifics)
- `direction` — one of: session | mud-in | mud-out | browser-in | browser-out | error
- `message` — JSON-string-encoded; contains escaped `\r\n` and ANSI `\u001b[...m` sequences

Output file format (per D-03 — KEEP original log format verbatim):
The extracted file is a byte-for-byte slice of source log lines between `[startTs]` and `[startTs + 30min]`; no re-encoding, no reformatting, no sanitization (D-05).
</interfaces>

    ```typescript
    // into .fixtures/mud-traffic-baseline.log for regression-oracle use.

    //   [<ISO timestamp>] session=<id> direction=<mud-in|mud-out|session|browser-*|error> message="<JSON>"
    //
    // Selection strategy (D-02): caller passes --start (ISO); script copies every
    // line whose timestamp is in [start, start + --minutes) to the output file.
    // Lines are written byte-for-byte from the source (no re-encoding per D-03).
    //
    // Run: bun run scripts/extract-baseline.ts --start 2026-04-15T09:30:00Z [--minutes 30] [--source PATH] [--out PATH]
    ```

    ### Imports (follow CONVENTIONS.md Import Organization):

    ```typescript
    import { createReadStream, mkdirSync, createWriteStream, existsSync } from "node:fs";
    import { dirname } from "node:path";
    import { createInterface } from "node:readline";
    ```

    ### Constants (module-scope, SCREAMING_SNAKE_CASE per CONVENTIONS.md):

    ```typescript
    const DEFAULT_SOURCE_LOG = "/var/log/bylins-bot/mud-traffic.log";
    const DEFAULT_OUTPUT_PATH = ".fixtures/mud-traffic-baseline.log";
    const DEFAULT_WINDOW_MINUTES = 30;
    const LOG_LINE_REGEXP = /^\[(?<ts>[^\]]+)\] session=(?<session>\S+) direction=(?<direction>\S+) message=/;
    ```

    Note regex stops at `message=` prefix — we do NOT need to parse the JSON body; we only need the timestamp.

    ### CLI arg parsing (hand-roll, no external dep):

    Parse `process.argv.slice(2)` into `{start: Date | null, minutes: number, source: string, out: string}`. Recognize:
    - `--start <ISO-8601>` — REQUIRED. Parse via `new Date(value)`. If `Number.isNaN(parsed.getTime())` → exit 1 with a message to stderr "extract-baseline: --start must be a valid ISO-8601 timestamp".
    - `--minutes <int>` — optional, default `DEFAULT_WINDOW_MINUTES` (30). Validate `Number.isFinite(n) && n > 0 && n <= 240`; if invalid → exit 1 with message "extract-baseline: --minutes must be a positive integer &le; 240".
    - `--source <path>` — optional, default `DEFAULT_SOURCE_LOG`.
    - `--out <path>` — optional, default `DEFAULT_OUTPUT_PATH`.

    If `--start` missing → print usage (copy of header's "Run:" line) to stderr and exit 1.

    ### Core function:

    ```typescript
    async function extractWindow(params: {
      source: string;
      out: string;
      start: Date;
      endExclusive: Date;
    }): Promise<{ linesRead: number; linesWritten: number }> {
      // 1. Ensure output dir exists (mkdirSync(dirname(out), {recursive: true}))
      // 2. createReadStream(source) -> readline interface (line-by-line, handles 756MB safely)
      // 3. createWriteStream(out) in "w" mode
      // 4. For each line: match LOG_LINE_REGEXP; if no match, skip (malformed line);
      //    otherwise parse ts via new Date(match.groups.ts);
      //    if ts >= start && ts < endExclusive -> write(line + "\n");
      //    if ts >= endExclusive -> break (lines are time-ordered, can stop early)
      // 5. Close write stream; return counters
    }
    ```

    Critical — stream semantics:
    - Use `createInterface({ input: createReadStream(source) })` to iterate lines — this handles the 756MB file without loading it into memory.
    - Do NOT `readFileSync` (will OOM on 756MB).
    - Early-break via `rl.close()` is fine because lines are time-ordered (logEvent appends monotonically).
    - When writing, use `writeStream.write(line + "\n")` — readline strips newlines; add back exactly one `\n`.

    ### Error handling (narrow unknown per CONVENTIONS.md):

    Wrap the `await extractWindow(...)` call in try/catch:
    ```typescript
    try {
      const { linesRead, linesWritten } = await extractWindow({ source, out, start, endExclusive });
      process.stdout.write(`extract-baseline: read ${linesRead} lines, wrote ${linesWritten} to ${out}\n`);
      if (linesWritten === 0) {
        process.stderr.write(`extract-baseline: WARNING no lines matched window [${start.toISOString()}, ${endExclusive.toISOString()})\n`);
        process.exit(2);
      }
      process.exit(0);
    } catch (error: unknown) {
      const message = error instanceof Error ? error.message : "unknown error";
      process.stderr.write(`extract-baseline: ${message}\n`);
      process.exit(1);
    }
    ```

    Exit codes:
    - `0` — success, non-zero lines written
    - `1` — CLI usage error or stream/fs failure
    - `2` — window produced zero matching lines (almost certainly a bad `--start`)

    ### Output type signatures (CONVENTIONS.md: explicit return types on exports):

    Keep `extractWindow` non-exported (script is meant to be run, not imported). Top-level `await` is fine (Bun / ESM supports it).

    ### No-Op on sanitization (D-05):

    Do NOT strip passwords, URLs, or any content. D-05: "Sanitization не требуется по решению пользователя".

    ### Pattern fidelity (anti-regex-drift, PITFALLS Pitfall 3):

    Use `LOG_LINE_REGEXP` EXACTLY as written above — no extra flags, no alterations. The regex is anchored at `^` and non-greedy until `message=`. Do not add `u`, `g`, `m`, or any flag.
