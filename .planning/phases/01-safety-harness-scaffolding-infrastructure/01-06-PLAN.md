**Phase 1 scope clarification (CRITICAL — avoid over-engineering):**

In Phase 1 there are NO controllers subscribed to the bus (D-29 — ports and bus are pre-wired, not yet consumed). So Phase 1's `snapshots/replay-before.jsonl` records only:
1. Every `bus.emit({kind:"mud_text_raw", text:...})` issued by the harness as it walks the fixture.
2. Every event produced by `feedText(parserState, chunk)` (which is also separately captured by Plan 05's parser-snapshot — intentional redundancy; the two harnesses are independent oracles).
3. Every `mapStore.<method>(args)` call — in Phase 1, **zero** such calls, because no controller consumes parser events to upsert rooms yet. The mock MapStore exists to be wired once Phase 2 extractions begin.
4. Every fake-clock tick — in Phase 1 no controller schedules a timer, so zero tick entries.

The initial `replay-before.jsonl` committed after Phase 1 will therefore contain ONLY bus emits + parser events. As Phase 2 subscribes controllers to the bus, the recorded surface grows. The first Phase-2 extraction PR will need to re-seed `replay-before.jsonl` because introducing a new subscriber legitimately changes the recorded sequence. This pattern is documented in Plan 07's playbook.

**Acceptance principle:** Plan 06 ships the harness tooling. Plan 06 does NOT commit `snapshots/replay-before.jsonl` — that is a developer-run seeding step. Plan 06's success criterion is "the harness runs, produces a non-empty transcript from the fixture, and diffs cleanly against itself".

### JSONL transcript format (per D-06, D-08):

Every recorded side-effect is one JSON-line with shape `{seq: number, kind: string, ...}`:

```jsonl
{"seq":0,"kind":"bus.emit","event":{"kind":"mud_text_raw","text":"..."}}
{"seq":1,"kind":"parser.events","chunkIndex":0,"events":[]}
{"seq":2,"kind":"bus.emit","event":{"kind":"mud_text_raw","text":"..."}}
{"seq":3,"kind":"parser.events","chunkIndex":1,"events":[{"kind":"room","room":{...}}]}
// ... when Phase 2 wires controllers:
{"seq":4,"kind":"mapStore.upsertRoom","args":[5321,"The Training Ground",["n","e"],[]]}
{"seq":5,"kind":"mudCommandSink.send","args":["вс","triggers"]}
{"seq":6,"kind":"timer.schedule","delayMs":1500,"tag":"loot-sort"}
{"seq":7,"kind":"timer.fire","tag":"loot-sort","atVirtualMs":1730000001500}
```

`seq` is a monotonic counter to make reordering bugs obvious in the diff. `kind` is a dot-separated namespace (`bus.emit` / `parser.events` / `mapStore.<method>` / `mudCommandSink.send` / `broadcaster.broadcast` / `timer.schedule` / `timer.fire` / `timer.clear`).

### Fake clock shape (CONTEXT D-10):

```typescript
export interface FakeClockHandle {
  now: NowProvider;
  timer: TimerProvider;
  /** Advance virtual time to `atMs`, firing every scheduled callback whose fire time <= atMs in insertion order for equal fire times. */
  advanceTo(atMs: number): void;
  /** Currently-scheduled pending timers (for transcript emission). */
  pending(): ReadonlyArray<{ id: number; fireAt: number; kind: "timeout" | "interval" }>;
}

export function createFakeClock(seedMs: number, sink: TranscriptSink): FakeClockHandle;
```

Every call to `timer.setTimeout` / `setInterval` / `clearTimeout` / `clearInterval` emits a transcript entry via the injected sink. When `advanceTo(t)` fires a timer, it emits a `timer.fire` entry BEFORE invoking the callback (so ordering in the transcript mirrors real-time dispatch).

Seed: `Date.parse(firstBaselineTimestamp)` — the first `[ISO]` in the fixture. Plan 06 reads that timestamp and passes it to `createFakeClock`.

    ### Factory

    ```typescript
    interface ScheduledTimer {
      id: number;
      fireAt: number;
      fn: () => void;
      kind: "timeout" | "interval";
      intervalMs?: number;
      cancelled: boolean;
    }

    export function createFakeClock(seedMs: number, sink: TranscriptSink): FakeClockHandle {
      let currentMs = seedMs;
      let nextId = 1;
      const queue: ScheduledTimer[] = [];

      function scheduleEmit(kind: "timeout" | "interval", id: number, delayMs: number): void {
        sink.emit({ kind: `timer.schedule-${kind}`, id, delayMs, atVirtualMs: currentMs });
      }

      const now: NowProvider = {
        now: () => currentMs,
      };

      const timer: TimerProvider = {
        setTimeout(fn: () => void, ms: number): TimerHandle {
          const id = nextId++;
          queue.push({ id, fireAt: currentMs + ms, fn, kind: "timeout", cancelled: false });
          scheduleEmit("timeout", id, ms);
          return id as unknown as TimerHandle;
        },
        clearTimeout(handle: TimerHandle): void {
          const id = handle as unknown as number;
          const entry = queue.find((t) => t.id === id && !t.cancelled);
          if (entry) {
            entry.cancelled = true;
            sink.emit({ kind: "timer.clear-timeout", id, atVirtualMs: currentMs });
          }
        },
        setInterval(fn: () => void, ms: number): IntervalHandle {
          const id = nextId++;
          queue.push({ id, fireAt: currentMs + ms, fn, kind: "interval", intervalMs: ms, cancelled: false });
          scheduleEmit("interval", id, ms);
          return id as unknown as IntervalHandle;
        },
        clearInterval(handle: IntervalHandle): void {
          const id = handle as unknown as number;
          const entry = queue.find((t) => t.id === id && !t.cancelled);
          if (entry) {
            entry.cancelled = true;
            sink.emit({ kind: "timer.clear-interval", id, atVirtualMs: currentMs });
          }
        },
      };

      function advanceTo(atMs: number): void {
        if (atMs < currentMs) {
          return; // never move backwards
        }
        // Fire eligible timers in fireAt ascending order; for ties, insertion order (preserved in queue).
        // Recompute eligibility after each fire since callbacks may schedule more timers.
        while (true) {
          const ready = queue.find((t) => !t.cancelled && t.fireAt <= atMs);
          if (!ready) break;
          currentMs = ready.fireAt;
          sink.emit({ kind: "timer.fire", id: ready.id, atVirtualMs: currentMs, timerKind: ready.kind });
          if (ready.kind === "interval" && ready.intervalMs !== undefined) {
            ready.fireAt = currentMs + ready.intervalMs;
          } else {
            ready.cancelled = true;
          }
          ready.fn();
        }
        currentMs = atMs;
      }

      function drain(): void {
        while (queue.some((t) => !t.cancelled)) {
          const next = queue
            .filter((t) => !t.cancelled)
            .reduce((a, b) => (a.fireAt <= b.fireAt ? a : b));
          advanceTo(next.fireAt);
        }
      }

      return {
        now,
        timer,
        advanceTo,
        drain,
        nowMs: () => currentMs,
      };
    }

    export function createFakeNowProvider(seedMs: number): NowProvider {
      let currentMs = seedMs;
      return {
        now: () => currentMs,
      };
    }

    export function createFakeTimerProvider(): TimerProvider {
      // Convenience standalone for callers that want just the TimerProvider shape without a sink.
      // Production-replay callers should use createFakeClock (which wires both under one virtual time).
      throw new Error("createFakeTimerProvider: callers should use createFakeClock for time-integrated scheduling.");
    }
    ```

    ### Main pipeline

    ```typescript
    async function generateTranscript(fixturePath: string, outPath: string): Promise<{ chunks: number; malformed: number }> {
      mkdirSync(dirname(outPath), { recursive: true });

      const seedMs = (await extractFirstTimestamp(fixturePath)) ?? 0;
      const { sink, entries } = createSink();
      const clock = createFakeClock(seedMs, sink);
      const mapStore = createMockMapStore({ sink });
      // Plan 1 note: mapStore is intentionally unused by any subscriber yet — D-29.
      // It's wired so Phase 2 extractions snap in via composition without reshaping the harness.
      void mapStore;

      const bus = createMudBus({
        onError: (message: string) => { sink.emit({ kind: "bus.error", message }); },
      });

      const parserState = createParserState();

      let chunkIndex = 0;
      let malformed = 0;

      const rl = createInterface({ input: createReadStream(fixturePath) });
      let lineNumber = 0;

      for await (const line of rl) {
        lineNumber++;
        if (line.length === 0) continue;
        const match = LOG_LINE_REGEXP.exec(line);
        if (!match?.groups) {
          malformed++;
          process.stderr.write(`replay-harness: WARNING skipping malformed line ${lineNumber}\n`);
          continue;
        }
        const direction = match.groups["direction"];
        if (direction !== TARGET_DIRECTION) continue;
        const ts = match.groups["ts"];
        const messageLiteral = match.groups["message"];
        if (ts === undefined || messageLiteral === undefined) continue;

        // Advance virtual time to this chunk's timestamp BEFORE emitting — drains any
        // pending timers scheduled by prior chunks. In Phase 1 no timers are scheduled,
        // so advanceTo just moves the clock. Phase 2 timer-driven controllers engage here.
        const atMs = Date.parse(ts);
        if (!Number.isNaN(atMs)) {
          clock.advanceTo(atMs);
        }

        let chunkText: string;
        try {
          chunkText = JSON.parse(messageLiteral) as string;
        } catch {
          malformed++;
          process.stderr.write(`replay-harness: WARNING skipping line ${lineNumber} — message not valid JSON\n`);
          continue;
        }
        if (typeof chunkText !== "string") {
          malformed++;
          continue;
        }

        sink.emit({ kind: "bus.emit", event: { kind: "mud_text_raw", text: chunkText } });
        bus.emit({ kind: "mud_text_raw", text: chunkText });

        const events = feedText(parserState, chunkText);
        sink.emit({ kind: "parser.events", chunkIndex, events });

        chunkIndex++;
      }

      // Drain any timers the last chunk scheduled.
      clock.drain();

      writeFileSync(outPath, entries.join("\n") + (entries.length > 0 ? "\n" : ""));
      return { chunks: chunkIndex, malformed };
    }
    ```

    Key design notes:
    - The harness emits a `{kind:"bus.emit", event:{...}}` transcript entry BEFORE calling `bus.emit(...)`. Phase 1 bus has no subscribers, so the real `bus.emit` is a pure no-op from the transcript's perspective. Emitting the transcript entry first preserves the "what the harness attempted" story even if Phase 2 adds subscribers whose own transcript entries come AFTER. The sequence is: `bus.emit` (harness intent) → any subscriber side-effects (in future phases) → `parser.events`.
    - `clock.advanceTo(timestamp)` runs BEFORE bus emit. This makes timers scheduled by prior chunks fire at their real-time relative position in the transcript, rather than all bunched at the end.
    - `mapStore` is constructed but intentionally unused in Phase 1 — `void mapStore` marks this to TypeScript without warning (verbatimModuleSyntax-safe). This ensures the harness "compiles with full pipeline" from Phase 1 onwards; Phase 2 just wires the subscriber that consumes events → calls mapStore.

    function diffSnapshots(beforePath: string, afterPath: string): { equal: boolean; firstDiff: string | null } {
      if (!existsSync(beforePath)) {
        return { equal: false, firstDiff: `replay-harness: before file missing at ${beforePath} — run --write-initial first` };
      }
      const beforeLines = readFileSync(beforePath, "utf8").split("\n");
      const afterLines = readFileSync(afterPath, "utf8").split("\n");
      if (beforeLines.length !== afterLines.length) {
        return { equal: false, firstDiff: `length mismatch: before=${beforeLines.length} lines, after=${afterLines.length} lines` };
      }
      for (let i = 0; i < beforeLines.length; i++) {
        if (beforeLines[i] !== afterLines[i]) {
          return { equal: false, firstDiff: `line ${i + 1} differs:\n  before: ${beforeLines[i]}\n  after:  ${afterLines[i]}` };
        }
      }
      return { equal: true, firstDiff: null };
    }
